# MyInvestShadow

MyInvestShadow 是一个本地影子账户 Web 系统，只消费两类已有研究结果：

- 市场结果：`https://market.okbbc.com//api/research/latest`
- 主线结果：`https://theme.okbbc.com/api/latest`

系统不做市场研究，不做主线研究，不连接真实下单，也不读取或对照真实持仓。它根据收盘基准日生成影子目标仓位，保存净值曲线，并把组合拆成核心仓位、主线仓位、主题仓位、防御仓位四层。

## 启动

```powershell
python .\scripts\run_web.py
```

默认地址：

```text
http://127.0.0.1:8013
```

## 对外 API

- `GET /api/index`：主页主要内容，包含概览、四仓位汇总、净值曲线、目标仓位和数据状态。
- `GET /api/latest`：最新影子组合完整状态，包含更完整的运行记录和调试字段。

## 本地运维接口

- `POST /api/run/daily`：立即拉取最新研究结果并生成影子仓位。
- `GET /api/nav`：资金净值曲线。
- `GET /api/allocations/latest`：最新影子目标仓位。

## 调仓原则

- 市场接口的权益仓位区间和市场分数决定主动仓位和防御仓位。
- 核心仓位使用市场宽基表现合成底仓，不跟随单条主线频繁变化。
- 主线仓位只使用“主线确认”与“次主线/强修复”的一号代表 ETF。
- 主题仓位只使用最高质量观察线，低于最小仓位阈值则回到防御仓位。
- 弱势/退潮方向直接归零。
- 收盘价和日涨跌优先使用本地 `.env` 中的 Tushare token；不可用时回退到主线接口的 ETF 排名数据。
- 所有对外展示都使用比例，不展示资金总额、股数或真实盈亏金额。

## 自动刷新

服务启动后会先补一次最新收盘基准日。运行期间会在每个交易日 15:05 后定时检查并刷新；同一基准日更新净值点，不重复累加。
